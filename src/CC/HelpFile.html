Logging In

Before Logging in to CAPS an Employee needs to be a User in the system.

To become a User in CAPS the Employee must be loaded in the CDMC import file.

To Login, enter the EmployeeID (and User Type, if User Type drop-down list is visible) and enter the password "a". Then click Login.

This will all be automated when set-up on a network with windows authentication, based on the User Network ID.

On successful login the user will arrive at their home screen. This will differ depending on the type of user they are.

Adding New Users

All users are added via the CDMC load (Administration --> Load CDMC Data) automatically.

The Username is the EmployeeID field and the password is always "a" for all users.

When the application is set-up on a network the user will automatically be logged in using windows authentication.

To load the CDMC File:
  • 1. Go to the CDMC Load screen
  • 2. Click in the "CDMC Template Excel" button. This will open an Excel workbook with the field names in the first row
  • 3. Fill in the relevant fields and save the Excel workbook as an ".xls" file ONLY
  • 4. Make sure the worksheet is named "CDMCData" only
  • 5. Click on the "Browse.." button and select the file saved in step 4. above
  • 6. Click on the "Upload" button

Data loaded can then be viewed in the CDMC screen.

Applying for a Card

To Apply for a Credit Card:

  • 1. Go to the Home text in the left-hand side menu
  • 2. Click on the "Apply For A Card" button next to the type of card
  • 3. The Application screen will open where details can be reviewed
  • 4. All contact details can only be changed via the CDMC file
  • 5. Click on the "Submit Application" button which will open a new window to confirm details and display user specific details
  • 6. Click on the "Submit" button

The application will then appear in the Applications screens and on the home page of the user with the status "Submitted".

Credit Card Application Approval Process:

  • 1. Employee Submits their Credit Card Application (status = "Submitted")
  • 2. GCFO receives application which must be Approved or Rejected (Status = "GCFO Approved" or "Rejected")
  • 3. DCC receives application which must be Approved or Rejected (Status = "Added to CS" or "Rejected by DCC")
  • 4. Application is added to CS file to be sent to Diners
  • 5. Application is loaded in CS From Diners file
  • 6. Application is given a card number and a new Card appears under "My Cards" on the Home screen and in Card screens

Searching for an Employee

To search for an Employee or card:

  • 1. Enter any part of the First Name, Last Name or EmployeeID in the "Search for.." text field in the top menu bar and press the enter key
  • 2. Click on the "Employee History" text in the left-hand-side menu and search for the Employee

Card and application screen can be filtered and searched.

Admin - Load CS From Diners

All Cards are created via the CS From Diners file load

To load the CS From Diners File:

  • 1. Export the existing CS To Diners file, by going to the CS To Diners screen and clicking "Export Batch" which will open an Excel workbook with the existing Applications
  • 2. Go to the CS From Diners screen (Administration --> CS From Diners)
  • 2. Click in the "CS From Diner Template Excel" button. This will open an Excel workbook with the field names in the first row
  • 3. Copy and paste the CS To Diners detail to the relevant fields and fill in the rest of the fields
  • 4. Save the Excel workbook as an ".xls" file ONLY
  • 5. Make sure the worksheet is named "CSFromDinersFile" only
  • 6. Click on the "Browse.." button and select the file saved in step 5. above
  • 7. Click on the "Upload" button

Card can then be viewed under "My Cards" on the Home screen and in Card screens
